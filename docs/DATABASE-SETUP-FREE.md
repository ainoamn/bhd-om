## الخيارات المجانية المناسبة

| المزود | الخطة المجانية | يكفي لـ 2000 عقار + 5000 مستخدم؟ | الترقية لاحقاً |
|--------|----------------|-----------------------------------|-----------------|
| **Neon** | 0.5 GB، مشروع واحد، Connection Pooling | ✅ نعم (البيانات أقل من 0.5 GB بكثير) | ترقية إلى خطط مدفوعة من نفس لوحة التحكم |

## خطوات إنشاء مشروع على Neon (مجاني)

1. ادخل إلى [neon.tech](https://neon.tech) وسجّل الدخول (GitHub أو البريد).

2. **إنشاء مشروع جديد** — اختر القيم التالية:

   | الحقل | اختر |
   |-------|------|
   | **Project name** | `bhd-om` (أو أي اسم) |
   | **Postgres version** | **17** (الأحدث، مستقر) |
   | **Cloud provider** | **AWS** |
   | **Region** | **AWS US East 1 (N. Virginia)** — يعمل جيداً مع Vercel. إن وُجد خيار أقرب لمنطقتك (مثل EU) يمكنك اختياره. |
   | **Enable Neon Auth** | **اتركه غير مفعّل** — نستخدم NextAuth وجدول User الخاص بنا. |

3. اضغط **Create project**.

4. بعد الإنشاء، من لوحة Neon:
   - ادخل إلى **Connection Details** أو **Dashboard**.
   - انسخ **Connection string**.
   - **مهم:** اختر الرابط من نوع **Pooled** (غالباً يظهر كـ "Pooled connection" أو ينتهي بـ `-pooler` في العنوان). هذا مطلوب لـ Vercel (Serverless).

5. الصق هذا الرابط في Vercel كقيمة للمتغير **`DATABASE_URL`** (Settings → Environment Variables)، ثم **Redeploy**.

## الترقية أو تغيير قاعدة البيانات لاحقاً (بدون فقدان بيانات)

الكود يعتمد على **رابط اتصال واحد** (`DATABASE_URL`) فقط. تغيير القاعدة = تغيير الرابط + نقل البيانات.

### تغيير إلى قاعدة بيانات جديدة (نفس PostgreSQL)

1. **إنشاء القاعدة الجديدة** (Neon خطة أعلى، أو Supabase، أو أي Postgres آخر).
2. **نسخ البيانات من القديمة إلى الجديدة:**
   - من جهازك، اتصل بالقاعدة **القديمة** وصدّر البيانات:
     ```bash
     pg_dump "رابط_القاعدة_القديمة" --no-owner --no-acl -F c -f backup.dump
     ```
   - استورد إلى القاعدة **الجديدة**:
     ```bash
     pg_restore -d "رابط_القاعدة_الجديدة" --no-owner --no-acl backup.dump
     ```
   - أو استخدم أدوات واجهة (مثل pgAdmin أو Neon Console) لتصدير/استيراد.
3. **تحديث الجداول إن لزم** (نفس الـ schema):
   ```bash
   DATABASE_URL="رابط_الجديدة" npx prisma db push
   ```
4. في **Vercel** غيّر قيمة **`DATABASE_URL`** إلى رابط القاعدة الجديدة ثم **Redeploy**.

لا حاجة لتعديل الكود؛ فقط تغيير `DATABASE_URL` ونقل البيانات يضمن عدم فقدان البيانات عند الترقية أو تغيير المزود.
