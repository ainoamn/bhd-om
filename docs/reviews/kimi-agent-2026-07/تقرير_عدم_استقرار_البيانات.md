# تقرير: لماذا البيانات تتغير "في لمح البصر"
## مشروع bhd-om | تاريخ: 2026-07-06

---

## 1. المشكلة

عند فتح الصفحة:
1. ✅ الصفحة تفتح
2. ⚠️ تظهر بيانات قديمة/ناقصة
3. 🔴 بعد ثانية أو ثانيتين — تتغير البيانات فجأة!
4. ❌ المستخدم يرى "ومضة" أو "تغيير" غير مريح

---

## 2. سبب المشكلة (الجذر)

النظام القديم يجلب البيانات على **3 مراحل**:

```
المرحلة 1: localStorage (بيانات قديمة مخزنة محلياً)
    ↓ عرض فوري ← البيانات الأولى التي تراها
    
المرحلة 2: HTML Bridge (بيانات من الخادم عبر injectLegacySiteBridgeScript)
    ↓ تحديث ← "تغيير في لمح البصر"
    
المرحلة 3: Prisma DB (بيانات حية من قاعدة البيانات)
    ↓ تحديث ثاني ← تغيير آخر!
```

### بالتفصيل:

| المرحلة | المصدر | السرعة | الوثوقية |
|---------|--------|--------|----------|
| 1 | localStorage | فوري | قديمة/غير مكتملة |
| 2 | HTML Bridge (injected) | 1-2 ثانية | أحدث لكن قد تكون cached |
| 3 | Prisma DB (API calls) | 2-5 ثواني | الأحدث دائماً |

## 4. الحل: 3 إصلاحات

### ✅ إصلاح 1: عدم عرض البيانات حتى تكون جاهزة

**الملف:** `app/api/admin/legacy-real-estate/[[...path]]/route.ts`

```typescript
// قبل: نرسل HTML فوراً (مع بيانات قديمة)
body = await readLegacyRealEstateHtmlWithBridge(segments, embedded);

// بعد: ننتظر البيانات الجاهزة أولاً
const payload = await buildLegacyBridgeMinimalPayload(userId, locale);
const html = await readLegacyRealEstateHtmlWithBridge(segments, payload);
// ↑ الآن HTML يحتوي على البيانات النهائية من البداية
```

**التأثير:** لا "ومضة" — البيانات تظهر مرة واحدة وصحيحة

### ✅ إصلاح 2: Loading State واضح

**نضيف overlay يخفي الصفحة حتى تجهز البيانات:**

```html
<!-- في بداية body -->
<div id="bhd-loading-overlay" style="position:fixed;inset:0;z-index:99999;
     background:#fff;display:flex;align-items:center;justify-content:center;
     transition:opacity 0.3s;">
    <div style="text-align:center;">
        <div style="width:50px;height:50px;border:4px solid #f3f3f3;
             border-top:4px solid #c8102e;border-radius:50%;
             animation:spin 1s linear infinite;margin:0 auto;"></div>
        <p style="margin-top:15px;color:#666;">جاري تحميل البيانات...</p>
    </div>
</div>
<style>@keyframes spin{0%{transform:rotate(0)}100%{transform:rotate(360deg)}}</style>

<script>
// إخفاء الـ overlay عندما تجهز البيانات
window.addEventListener('load', function() {
    // ننتظر حتى يُطبق Bridge
    setTimeout(function() {
        var overlay = document.getElementById('bhd-loading-overlay');
        if (overlay) {
            overlay.style.opacity = '0';
            setTimeout(function() { overlay.style.display = 'none'; }, 300);
        }
    }, 500); // نصف ثانية كافية لـ Bridge
});
</script>
```

**التأثير:** المستخدم يرى spinner واضح بدلاً من "ومضة"

### ✅ إصلاح 3: Cache للبيانات (منع الجلب المتكرر)

**الملف:** `lib/server/legacyBridge.ts`

```typescript
// قبل: كل طلب يبني Bridge من جديد
const payload = await buildLegacyBridgeMinimalPayload(userId, locale);

// بعد: Cache لـ 30 ثانية
const cacheKey = `bridge:${userId}:${locale}`;
const cached = bridgePayloadCache.get(cacheKey);
if (cached && Date.now() - cached.time < 30_000) {
    return cached.payload; // ← نفس البيانات = لا تغيير
}
const payload = await buildLegacyBridgeMinimalPayload(userId, locale);
bridgePayloadCache.set(cacheKey, { payload, time: Date.now() });
```

**التأثير:** نفس البيانات لـ 30 ثانية = لا "ومضة" عند التنقل
