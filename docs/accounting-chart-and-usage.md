# دليل الحسابات واستخدامها — النظام المحاسبي

## الحسابات المخصصة حسب نوع النشاط

### 1. إيرادات الاشتراكات (الباقات) — **4250**
- **الاسم (عربي):** إيرادات الاشتراكات (الباقات)
- **الاسم (إنجليزي):** Subscription Revenue
- **الاستخدام:** كل دفعة من المستخدم مقابل الباقة/الاشتراك (خطط الاشتراك).
- **مصادر القيود:**
  - إتمام الدفع من صفحة «حسابي» (ترقية/تنزيل الباقة).
  - مزامنة سجل الاشتراكات (SubscriptionHistory) عند فتح لوحة المحاسبة.
- **قيد محاسبي نموذجي:** مدين الصندوق (1000) / دائن إيرادات الاشتراكات (4250).

---

### 2. إيرادات العقارات والإيجار — **4000**
- **الاسم (عربي):** إيرادات العقارات والإيجار
- **الاسم (إنجليزي):** Property & Rent Revenue
- **الاستخدام:** كل إيصال حجز وحدة عقارية (دفعات الحجوزات).
- **مصادر القيود:**
  - إتمام حجز وحدة مع دفع من صفحة الحجز.
  - مزامنة الحجوزات المدفوعة (BookingStorage) عند فتح لوحة المحاسبة.
- **قيد محاسبي نموذجي:** مدين الصندوق (1000) / دائن إيرادات العقارات والإيجار (4000).

---

### 3. الصندوق — **1000**
- **الاستخدام:** استلام النقد (دفعات الاشتراكات والحجوزات تُسجّل مديناً هنا).

### 4. البنوك — **1100**
- **الاستخدام:** الحسابات البنكية عند اختيار الدفع بالتحويل.

---

## مبدأ عمل النظام المحاسبي

1. **قيد مزدوج:** كل معاملة تُسجّل بمدين يساوي الدائن.
2. **مستند → قيد:** لكل إيصال (دفع اشتراك أو حجز) يُنشأ مستند محاسبي وقيد يومية مرتبط.
3. **فصل المصادر:**
   - الاشتراكات (المستخدمين/الباقات) → حساب **4250** فقط.
   - العقارات (حجوزات الوحدات) → حساب **4000** فقط.
4. **المزامنة التلقائية:** عند فتح لوحة المحاسبة يُشغّل النظام مزامنة الحجوزات المدفوعة ومزامنة سجل الاشتراكات، ثم يُنشئ الإيصالات والقيود الناقصة في قاعدة البيانات.

---

## إنشاء الحسابات تلقائياً

- **إيرادات الاشتراكات (4250):** تُنشأ عند أول استخدام عبر `ensureSubscriptionRevenueAccount()`.
- **إيرادات العقارات والإيجار (4000):** تُنشأ عند أول استخدام عبر `ensurePropertyRentRevenueAccount()`.
- دليل الحسابات الكامل (الصندوق، البنوك، الإيرادات، المصروفات، إلخ) يُنشأ عند أول تحميل للوحة المحاسبة إذا كانت الجداول فارغة.
