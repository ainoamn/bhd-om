# الجزء الأول: نظام المحاسبة (Accounting System)

---

## 1. نظرة عامة

نظام المحاسبة في bhd-om هو **نظام ERP متكامل** يتبع معايير **IFRS** و**COSO Framework** مع دعم **القيمة المضافة العمانية (VAT 5%)** والضرائب العمانية.

### حجم النظام:
| البند | العدد |
|-------|-------|
| ملفات TypeScript | **44 ملف** |
| أنواع حسابات | 5 (أصل/التزام/حقوق ملكية/إيراد/مصروف) |
| أنواع المستندات | 10 (فاتورة/إيصال/عروض سعر/قيد يومية/...) |
| أنواع التقارير | 6+ (ميزانية عمومية/قائمة دخل/تدفقات نقدية/ضريبة/تقدم في العمر/مقارنة فترات) |
| أدوار RBAC | 4 (محاسب/معتمد/مراجع/مدير) |

## 3. آلية عمل النظام

### 3.1 دورة حياة القيد المحاسبي (Journal Entry Lifecycle)

```
إنشاء قيد ← التحقق ← اعتماد ← ترحيل ← تدقيق
   │           │         │        │        │
   ▼           ▼         ▼        ▼        ▼
DRAFT      validate   APPROVED  POSTED   AUDIT_LOG
           (balance)           (ledger)
```

**قاعدة ذهبية:** "لا عملية بدون قيد | لا قيد بدون مستند | لا تعديل بدون أثر تدقيقي"

### 3.2 التحقق من القيد (Validation)

| # | التحقق | الوصف |
|---|--------|-------|
| 1 | طرفين على الأقل | مدين + دائن كحد أدنى |
| 2 | التوازن | مجموع المدين = مجموع الدائن (±0.001) |
| 3 | عدم السالب | المبالغ يجب أن تكون موجبة |
| 4 | طرف واحد | كل سطر إما مدين أو دائن (وليس الاثنين) |
| 5 | الفترة المغلقة | لا يمكن الترحيل لفترة مغلقة |

### 3.3 أنواع الحسابات (Chart of Accounts)

| الكود | النوع | أمثلة |
|-------|-------|-------|
| 1xx | أصول (ASSET) | النقدية/الذمم/المخزون/الأصول الثابتة |
| 2xx | التزامات (LIABILITY) | الموردون/القروض/المستحقات |
| 3xx | حقوق ملكية (EQUITY) | رأس المال/الأرباح المحتجزة |
| 4xx | إيرادات (REVENUE) | إيرادات الإيجار/المبيعات |
| 5xx | مصروفات (EXPENSE) | الرواتب/الإيجار/الصيانة |

### 3.4 أنظمة الضرائب المدعومة

| الضريبة | النسبة | الاستخدام |
|---------|--------|-----------|
| VAT (ضريبة القيمة المضافة) | 5% | المدخلات/المخرجات |
| Income Tax (ضريبة الدخل) | شرائح (0%-25%) | على الأفراد والشركات |
| Withholding Tax (خصم عند المنبع) | 5%-10% | الخدمات المهنية والإيجارات |

### 3.5 صلاحيات الوصول (RBAC)

| الدور | إنشاء قيود | اعتماد | إلغاء | إغلاق فترات | تقارير | تدقيق |
|-------|-----------|--------|-------|-------------|--------|-------|
| ACCOUNTANT (محاسب) | ✅ | ❌ | ❌ | ❌ | ✅ | ❌ |
| APPROVER (معتمد) | ✅ | ✅ | ✅ | ❌ | ✅ | ❌ |
| AUDITOR (مراجع) | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| ADMIN (مدير) | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

### ⚠️ نقاط الضعف والتحسين

| # | المشكلة | الخطورة | الحل |
|---|---------|---------|------|
| 1 | `core.ts` يستخدم Prisma لكن `journalEngine.ts` يستخدم localStorage | **حرجة** | توحيد المصدر (Prisma فقط) |
| 2 | `postJournalEntry` في `core.ts` معطل (تعليقات) | **حرجة** | إكمال ترحيل الأرصدة |
| 3 | `balance` في `AccountingAccount` غير مستخدم في Prisma | عالية | إضافة حقل balance أو حسابه ديناميكياً |
| 4 | التخزين المزدوج (localStorage + Prisma) | عالية | إلغاء localStorage |
| 5 | `taxEngine.ts` يخزن في `KEYS.JOURNAL` (خطأ) | عالية | إنشاء تخزين منفصل للضرائب |
| 6 | لا يوجد حساب تراكمي للأرصدة | عالية | إضافة Running Balance |
| 7 | 44 ملف مفرق | متوسطة | تجميع في 10-15 ملف |
| 8 | التعليقات العربية في `core.ts` | منخفضة | توثيق JSDoc |
| 9 | لا يوجد اختبارات وحدات للمحاسبة | عالية | إضافة Jest tests |

## 5. خارطة طريق التطوير

### المرحلة 1: الإصلاحات الحرجة (أسبوع 1-2)

```typescript
// 1. توحيد التخزين في Prisma فقط
// في journalEngine.ts - استبدل:
const entries = getStored<JournalEntry[]>(STORAGE_KEYS.JOURNAL);
// بـ:
const entries = await prisma.accountingJournalEntry.findMany();

// 2. إكمال postJournalEntry في core.ts
// إضافة تحديث الرصيد:
await tx.accountingAccount.update({
  where: { id: line.accountId },
  data: { balance: { increment: adjustment } }
});

// 3. إضافة حقل balance في Prisma schema:
// balance Float @default(0)
```

### المرحلة 2: التحسينات (أسبوع 3-4)

| التحسين | الوصف | التأثير |
|---------|-------|---------|
| Running Balance | رصيد تراكمي لكل حساب | تتبع فوري للأرصدة |
| Multi-Currency | دعم USD/EUR/OMR | توسع دولي |
| Cost Centers | مراكز التكلفة | تتبع المشاريع |
| Budgeting | الميزانية مقابل الفعلي | الرقابة المالية |
| Automated Entries | قيود تلقائية (إهلاك/مستحقات) | تقليل العمل اليدوي |
